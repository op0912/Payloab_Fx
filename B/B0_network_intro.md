# BLOC B — Network introduction

## Context
This is my first networking block.

The goal is to build a solid foundation before going deeper into offensive security.

---

## Objective
Understand how machines communicate over a network in a way that allows observation, analysis, and future exploitation.

---

## Certification Goal
This block aligns with the fundamentals required for the CompTIA Network+ certification.

---

## Approach
Focus on understanding real network behavior, not memorization.

Build an offensive mindset from the start by identifying how communication works and where it can be attacked.

---

## Structure

This block is divided into 13 sub-blocks.

Each sub-block covers a key part of networking, following how data moves and how systems interact.

Hands-on labs are integrated directly into each sub-block as observations and practical validation.

---

### B1 — Network Flow
Understand how a packet moves from one machine to another.

---

### B2 — Transport
Understand how communication happens between machines and programs.

---

### B3 — Services
Understand what services do (DNS, HTTP, DHCP).

---

### B4 — Ports & Exposure
Understand open ports and exposed services.

---

### B5 — Filtering & Control
Understand how firewalls allow or block communication.

---

### B6 — OSI Model
Use OSI as a reference to locate problems and understand layers.

---

### B7 — IPv4 Basics
Understand IP addressing and basic network structure.

---

### B8 — Network Equipment
Understand the role of routers, switches, and firewalls.

---

### B9 — WiFi & Physical Layer
Understand basic wireless and physical network concepts.

---

### B10 — Network Attacks
Understand common attacks related to networking.

---

### B11 — VLAN & Segmentation
Understand how networks are logically separated.

---

### B12 — Traffic Types
Understand how data is sent (unicast, broadcast, etc.).

---

### B13 — Network Topologies
Understand basic network structures.

---

## Labs (Integrated)

Labs are not separated from theory.

They are embedded within each sub-block to:
- observe real behavior
- validate understanding
- build practical intuition

Core lab activities include:

- Checking local network configuration (ipconfig)
- Testing connectivity (ping)
- Visualizing network paths (tracert)
- Observing active connections and ports (netstat)
- Resolving domain names (nslookup)
- Comparing HTTP vs HTTPS behavior
- Interacting with firewall rules (allow / deny)
- Observing real WiFi environment (frequency, security)
- Practicing with real platforms (TryHackMe)
- Identifying ports and services in real scenarios
- Analyzing IP types and ranges
- Completing exam-style questions (200+ target, 80%+ goal)
- Drilling port ↔ service recognition (instant reflex)

---

## End Goal

Be able to:

- understand how data moves across a network
- identify key components (IP, ports, services)
- recognize exposed services and attack surface
- explain basic network behavior and issues
- connect theory to real-world observations
